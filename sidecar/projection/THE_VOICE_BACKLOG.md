# THE VOICE — the backlog (rendering the instrument's truth into the TTY)

**This is not a punch list. It is the third layer of a self-proving instrument, told as
work.** Before any file:line below means anything, hold the North Star (§0) and the register
(§2) — because a string that passes every mechanical guard and still shows the operator the
*apparatus* instead of the *truth* is not done, and the guard cannot tell the difference.

> **The doc set, in one breath.** `NORTH_STAR.md` is *why the engine exists* (fidelity as a
> theorem it proves about itself). `THE_INSTRUMENT.md` is *what the operator experiences*
> (one essence, infinitely diggable; the newcomer is the power user). **`THE_VOICE.md` is the
> register — the law for every string you will write here; §2 distills it, but read the
> source before you start, and keep it open while you work.** `THE_STORYBOARD.md` is the
> surface (act × stream × outcome). `THE_CLI.md` is the command shape the voice speaks on (the

## 0 — The North Star (why a single raw line matters)

The engine is one **adjunction** between a logical Model and a physical Substrate
(`NORTH_STAR.md` §1):

```
Project : Model     ──►  Substrate     (emit)
Ingest  : Substrate ──►  Model         (read back)
Law     : Ingest ∘ Project = identity  (modulo named, declared erasures)
```

Everything the engine does is a corollary of that one law. The deepest thing it knows about
itself is that this round-trip is **faithful** — not trusted, *proven*, continuously, about
itself. The OutSystems estate's schema and data **mean exactly what the model says — provably,
in both directions, across time.** Fidelity is a theorem, not a property a human verifies.

A self-proving instrument is worth nothing if the operator cannot *read* the proof. So the
instrument has three layers (`THE_INSTRUMENT.md`): **Structure** (`View` — *what* is shown),
**Style** (`Theme` — *how it looks*), and **Voice** (*how it speaks* — the register that
renders the deep truth as the **essence**: the plain human finding a newcomer trusts on sight,
the formal proof one level beneath for whoever has the question). The Voice is this third
layer. Its job is to take the theorem — `residual ∅ · Ingest ∘ Project = id` — and say it as
*"Verified. The database matches the model."* The essence is the proof made kind; the proof is
the essence made rigorous; **the surface never changes between the newcomer and the master —
only the velocity.**

And the instrument is supposed to **disappear**: "What remains is the schema's truth and the
operator's hand on it… You are standing inside a system that already knows." **Every raw
render site in this backlog is a seam where the instrument fails to disappear** — a `%A` dump,
a `norm=` lead, a `REFUSED` shout, a leaked exit code on the statement line. The apparatus
shows through and the operator stops reading the truth and starts reading the tool. A `sprintf
"%A"` at the verdict is not a lint finding; it is the migration's *answer to "is it safe? did
it work?"* rendered as an internal DU dump, at the moment the operator most needs the essence.
**The file:line is where; the North Star is why; the register is how.**

## 1 — What "done" means (the bar the guard cannot see)

The `VoiceTotalityTests` banned-list guard is **necessary and not sufficient.** It catches a
pronoun, a shout, a `%A`. It cannot tell you whether the line is *true*, *grounded*, or
*kind*. A line is done when all of these hold:

1. **One register, two velocities.** The newcomer reads it as orientation; the master reads the
   *same line* as a glance already past. If it serves only one reader, it is not done.
2. **Essence on top, proof one level beneath.** The lead is a complete plain finding, readable
   aloud with no prior context. Notation (`‖δ‖`, exit codes, the commuting square, the gate
   label) lives in the substantiation, reachable, never on the statement line.
3. **The finding is asserted with grounded authority.** What the engine *proves* is stated
   plainly; the proof beneath earns it. Hedge only on genuine interpretation (a recommendation).
4. **It ends on the move.** The lever, the gate, the next command. Nothing terminates at "done";
   where nothing remains, the surface *says* that.
5. **The instrument disappears.** No internal vocabulary, no apparatus. `Country`, not
   `OS_KIND_Country`; the schema's truth, not the plumbing.

**The felt arc is the target, not the string** (`THE_VOICE.md` §7). Each workflow has a composed
shape — *alert → consenting → verified → trusting* for an in-place migrate. When you voice a
surface you place one frame in that arc; ask what the operator should *feel* one step before
and after, and whether your line carries them.

> **The acceptance question is not "does the guard pass?" It is: would the newcomer trust this
> on sight, would the master read it as a glance, and does the instrument disappear behind it?**

## 2 — The register in one screen (memorize before writing a string)

This distills `THE_VOICE.md` §1 + §2.2. **It is not a substitute for reading them** — but keep
it in view. Most of these are corrections you will otherwise re-introduce by instinct; the
operator settled them one at a time and will notice every relapse.

### The twelve rules (the floor)

1. **No pronouns** — no *I / we / you / your*. Trust comes from precision, not warmth.
2. **Direction by imperative** — the next move is a bare imperative: `Grant ALTER, then retry.`
3. **Legible statement, formal substantiation beneath** — no symbolic shorthand on the lead
   (`= → ∅ ‖·‖`), no fragments. Notation lives one level down.
4. **Verdicts are findings, asserted** — *safe · reversible · idempotent · matched* stated with
   authority; hedge only on genuine interpretation.
5. **The true verb** — `Drops · Deletes · Narrows · Rewrites`; never softened (*removes, cleans
   up*), never dramatized (*destroys*).
6. **Gentle and direct; never colloquial** — no casual idiom.
7. **Neutral reference to the estate** — *the model · the database · 300 tables*, never *your*.
8. **Ground every claim in its evidence** — assert the positive evidential claim. Forbidden: the
   antithesis tic (`X, not Y` — *"verified, not assumed"*, *"and that's real"*).
9. **Order by real structure** — perfect aspect for what is established; explicit consequence for
   what follows. No ambiguous habitual present.
10. **Name the exact referent** — the precise operation/predicate, never a loose paraphrase.
11. **Concrete definite subjects** — never a headless quantity (`No changes` → `The database is
    provably unchanged`).
12. **Stative, agentless voice** — report states/events, never actions performed. `Read the
    model — 300 tables` → `Model read complete — 300 tables`; `Widened the Email column` →
    `Email column widened`. *Exception:* a gerund naming a live activity in progress (`Reading
    the model`, `Checking the data`) is a state, kept.

### The banned list (§2.2) — never on any operator surface

Pronouns (*I, we, you, your*) · the antithesis tic (*verified, not assumed*; *that's real*) ·
euphemism (*removes, cleans up* for a drop) · drama (*destroy(s), fatal, abort, blast radius*) ·
figurative terms (*dig, diggable, green hush, jewel, open a lane*) · **system-shout as a lead**
(`REFUSED`, `ERROR`, `FAILED`) · jargon on the statement (*δ, norm, torsor, quotient, commuting
square, isomorphism, surrogate, episode*) · colloquialism (*oops, let's, hang on*) · leaked
internals (`OS_KIND_*`, raw `SsKey`, file paths where a name belongs, **exit codes on the
statement line**) · negation-as-headline (*"nothing destroyed"* → *"no removals"*).

### Off → on (the calibration that prevents drift — `THE_VOICE.md` §11)

| ✗ Off (the failure mode) | ✓ On (the gold standard) | Rule |
|---|---|---|
| `3 changes · 3 destroy structure` | `3 changes · 3 drops · review before applying` | 5 |
| `· OS_KIND_Country` | `· Country` | 2.1 lexicon |
| `‖δ‖ norm  ▲ 3` | `Total changes: 3` | 3 |
| `migrate refused — undeclared destructive change · exit 9` | `Cannot migrate yet: a database index would be dropped. Approve the removal to continue.` | §5/§10 |
| `nothing destroyed` | `no removals` | §2.2 |
| `Nothing changed — and that's real.` | `Verified. The database is provably unchanged.` | 8 |
| `Your data stayed put.` | `No rows were rewritten.` | 1 / 7 |
| `I touched exactly the 312 rows that changed.` | `312 rows changed — exactly those that differed, and no others.` | 1 / 12 |
| `Operation completed successfully.` | `Verified. The database matches the model.` | 4 / 11 |

**The lexicon translates, always** (§2.1): `Kind`→**the table's name**; `Attribute`→**the
column's name**; `‖δ‖`/norm/CDC count→**rows changed**; `Ingest∘Project = id`→**the database
matches the model**; canary→**self-check**; drift→**a divergence on the server**; gate→**paused
/ approval required**; episode→**this run / the record**.

## 7 — Disciplines (hold the vision and the register, not just the guard)

- **The acceptance bar is §1, the register is §2 — the guard is only the floor.** A guard-clean
  line that lands the operator nowhere in the felt arc is unfinished. When unsure, read
  `THE_VOICE.md` §11 (off → on) and §7 (the felt arc).
- **Derive, never invent** (`THE_VOICE.md` §15). Every string comes from a move (§4), gate (§5),
  proof (§6), error (§10), or config (§14) example — keep the register, fit the specifics. New
  operator word? Add it to `THE_VOICE.md` §2.1 first, then use it.
- **Codes never change, only copy.** The NDJSON event contract is the machine channel
  (`THE_VOICE_BUILD_MAP.md` §8.2 — DO-NOT-BREAK). No event moves; this is pure rendering.
- **Declare-at-site, harvest-centrally; `code ⇔ copy` + gate⇔copy totality.** Every new flat code
  enters the in-scope set as it lands; a new `GateLabel` without §5 copy fails the build.
- **IR grows under evidence.** Voice what an executor actually emits; never ahead of its events.
- **Pure-Core holds.** Operator prose never enters `Projection.Core`; `View`/`Surface`/`Voice`
  live in `Projection.Cli`.
